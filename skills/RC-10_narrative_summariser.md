# RC-10 — Narrative Summariser

## Purpose

Convert any structured agent output into a plain English narrative suitable for non-technical stakeholders. This skill makes complex supply chain data accessible to anyone.

---

## Owner

**Primary:** Communications & Reporting Agent

---

## When This Skill Is Used

- As needed, for any agent output that needs to be communicated to non-technical audiences
- Can be applied to any report or alert to produce a narrative version

---

## Inputs

1. **Structured agent output** — Any output from any agent that needs a narrative version

---

## Steps

1. **Identify the key message** — What is the single most important thing this output is saying?
2. **Translate technical content into plain English** — Replace jargon with everyday language, define any domain terms that must be used
3. **Structure as a readable narrative** — Short paragraphs, clear flow from headline to detail to action
4. **Maintain accuracy** — The narrative must be factually consistent with the structured output
5. **End with what happens next** — What should the reader do or expect?

---

## Output

- Plain English narrative version of the structured output
- Suitable for email, presentation, or executive briefing

---

## Notes

- The narrative should stand alone — a reader should not need to also read the structured output
- Quantify everything — "stock is low" becomes "stock at Depot A is at 3.2 weeks of supply, below the 6-week safety threshold"
- This is the most audience-sensitive skill — match the language to the reader
